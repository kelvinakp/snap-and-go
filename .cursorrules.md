# Snap and Go - Full Stack Cursor AI Rules

## Tech Stack Requirements
* [cite_start]Frontend: Next.js (App Router), TypeScript, Tailwind CSS, PWA configured[cite: 11, 8].
* [cite_start]Backend: Python, FastAPI, Uvicorn[cite: 12, 8].
* [cite_start]AI/Computer Vision: Ultralytics YOLOv8 Nano (`yolov8n.pt`)[cite: 14].
* [cite_start]Payment Processing: `pyzbar` (QR detection) and `promptparse` (EMVCo parsing).
* [cite_start]Database: Supabase (PostgreSQL)[cite: 13, 8].

## Design & UI Principles
* [cite_start]Mobile-First Approach: Optimize strictly for mobile browser viewports, ensuring flawless operation on iOS Safari and Android Chrome[cite: 35].
* Premium Aesthetic: Implement a clean, modern interface inspired by high-end e-commerce. Use generous whitespace, rounded corners (`rounded-2xl`), and subtle drop shadows.
* Tactile Feedback: Ensure all interactive elements use Tailwind's `active:` states (e.g., scale-down on tap).

## Code & Integration Conventions
* State Management (CRITICAL): The global store (Zustand/Context) must support an accumulative cart. The `addItemsToCart` function must process arrays returned from the backend, incrementing quantities for existing items or adding new ones.
* API Communication (Multipart/FormData): 
    * [cite_start]`/api/scan-cart`: Frontend uses `fetch` to send the captured camera frame as a `multipart/form-data` Blob[cite: 19].
    * `/api/verify-payment`: Frontend uses `fetch` to send the e-slip image AND the `expected_amount` (from cart state) as `multipart/form-data`.
* Strict Typing: Enforce TypeScript interfaces for both the frontend state and the expected JSON payloads from the FastAPI backend.
* Error Handling: Implement graceful fallbacks on the frontend for network timeouts, empty YOLO detections, or invalid/unreadable QR codes.
* Backend Structure: Keep the FastAPI logic modular. Separate the YOLO inference logic, the EMVCo parsing logic, and the route handlers.
* **CRITICAL OS DEPENDENCY:** The backend requires the `zbar` system library installed on the host machine to run `pyzbar` successfully.
